# Pilot Readiness Checklist

Use this before treating HanziBit as ready for real daily learner use.

## Product Focus

- [ ] learner navigation is simplified enough for a solo beginner
- [ ] teacher-only complexity stays out of the way unless relevant
- [ ] the dashboard clearly tells a new learner what to do next

## Onboarding

- [ ] sign up works cleanly
- [ ] sign in works cleanly
- [ ] first dashboard load is understandable for a beginner
- [ ] empty states give useful action guidance
- [ ] first study action is obvious

## Core Learner Loop

- [ ] Study Guide works reliably
- [ ] guided journal drafting works reliably
- [ ] first journal save works reliably
- [ ] flashcard review works reliably
- [ ] dashboard reflects progress after those actions

## Beginner Clarity

- [ ] copy is understandable without prior HSK/app knowledge
- [ ] no “phase/demo/internal” language leaks into learner surfaces
- [ ] loading states feel intentional
- [ ] error states are understandable

## Trust / Production Basics

- [ ] auth behavior is consistent
- [ ] billing/upgrade path is stable
- [ ] teacher/referral/admin features do not interfere with the learner path
- [ ] docs reflect the real app state

## First-Week Habit

- [ ] the learner can repeat the loop daily
- [ ] the app gives enough sense of momentum
- [ ] the app makes tomorrow’s next step feel obvious
