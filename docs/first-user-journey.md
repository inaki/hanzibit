# First User Journey

This document defines the intended first real learner journey for HanziBit.

The reference learner is:

- starting Chinese from zero
- using HanziBit without teacher support at first
- trying to build a daily habit

## Product promise

HanziBit should feel like:

- a clear daily path
- a place to notice Chinese in context
- a place to write something small every day
- a place to remember what was learned

Not:

- a complex school platform
- a dashboard of teacher/admin systems
- a tool that requires setup before it helps

## First-session journey

### Step 1: Sign up

The learner should:

- create an account quickly
- land in the notebook/dashboard

The product should:

- avoid showing unnecessary teacher complexity
- avoid empty, contextless screens

### Step 2: Understand Today

The learner should understand:

- Today’s Practice is the main entry point
- the app wants them to:
  - study
  - write
  - review

The product should:

- explain the daily loop in beginner language
- show one obvious first action

### Step 3: Open one study item

The learner should:

- open a simple Study Guide word
- see:
  - the word
  - pinyin
  - meaning
  - one tiny example

The product should:

- keep the content compact
- avoid overwhelming the learner with too many panels at once

### Step 4: Write one guided response

The learner should:

- go from study item to journal
- write one very short response

The product should:

- prefill enough context
- not require the learner to invent from scratch
- keep annotation optional/helpful, not intimidating

### Step 5: Review a few flashcards

The learner should:

- do a tiny review session

The product should:

- make flashcards feel connected to what was just studied
- avoid turning review into a separate disconnected mode

### Step 6: Leave with a reason to return

The learner should leave knowing:

- what they did
- that it counts
- what to do next time

## First-week journey

### Day 1

- learn one or two words
- write one tiny response
- review a tiny set of cards

### Day 2

- return to Today’s Practice
- recognize the same loop
- feel the app is already familiar

### Day 3

- understand that journal, study, and flashcards reinforce each other
- feel slight progress, not confusion

### Day 7

- have a small streak / rhythm
- have several entries
- have repeated exposure to a small HSK 1 set

## Key UX rules for this learner

### 1. Reduce noise

Default learner navigation should emphasize:

- Dashboard
- Daily Journal
- Study Guide
- Flashcards

Everything else is secondary.

### 2. Prefer guided over open-ended

For beginners:

- prompts should be tiny
- study items should be narrow
- actions should be obvious

### 3. Keep progress visible

A beginner needs to feel:

- “I did something real”
- “I can come back tomorrow”

### 4. Avoid jargon early

Do not assume the learner understands:

- HSK
- annotation markup
- guided response systems
- teacher/platform terminology

Those should be explained through use, not dumped upfront.

## Immediate product opportunities

Based on the current app, the next highest-value improvements for this journey are:

1. beginner-friendly onboarding copy
2. a clearer first dashboard state
3. simpler empty states
4. beginner study-guide content prioritization
5. first-week daily-flow shaping
