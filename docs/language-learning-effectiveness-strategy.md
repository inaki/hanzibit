# HanziBit Language Learning Effectiveness Strategy

## Purpose

This document translates current language-learning research and product ideas into concrete strategy for HanziBit.

It is intended to complement:

- `docs/product-evolution-strategy.md`
- `docs/referral-program-strategy.md`

The goal is to answer one product question clearly:

**How should HanziBit evolve if we want it to become a very efficient app for actually learning a language well?**

---

## Core Conclusion

The strongest direction for HanziBit is not to become a bigger content library or a heavier flashcard app.

The strongest direction is to become a **daily language practice system** built around this loop:

1. comprehensible input
2. noticing and annotation
3. retrieval and spaced review
4. pushed output
5. corrective feedback
6. repetition over time

This aligns both with current research and with HanziBit’s strongest existing product asset: the notebook.

---

## What HanziBit Already Does Well

The current product already supports part of an evidence-based learning loop:

- journal writing
- vocabulary in context
- interlinear glossing
- HSK-aligned study
- flashcard review
- progress tracking

This means HanziBit is not starting from zero.

The app already has a meaningful foundation for:

- output
- noticing
- retrieval
- structured progression

That is a strong base.

---

## Main Product Gap

The current app is still stronger at:

- writing
- review
- browsing HSK content

than it is at:

- delivering comprehensible input
- turning input into output
- giving corrective feedback
- guiding a complete daily practice routine

This is the main gap if HanziBit wants to become a highly effective language-learning system.

---

## Product Principle

HanziBit should be positioned and designed as:

**a system that turns input into output, and output into long-term retention**

That is more powerful and more differentiated than:

- notebook app
- flashcard app
- HSK reference app

---

## Learning Model HanziBit Should Optimize For

## 1. Comprehensible Input First

Learners need large amounts of understandable reading and listening at the right level.

This suggests HanziBit should support:

- graded reading passages
- short dialogues
- short listening passages with transcripts
- level-appropriate mini stories
- click-to-gloss support inside input content
- recommendations based on what the learner is likely to understand

### Product implication

The study guide should evolve from a vocabulary browser into a **comprehensible input hub**.

---

## 2. Noticing In Context

Input becomes more useful when the learner can notice vocabulary, grammar, and patterns without leaving the flow.

This suggests HanziBit should emphasize:

- inline annotation
- interlinear glossing
- quick word and phrase lookup
- pattern highlighting
- one-click save to review

### Product implication

The app should make it easy for learners to move directly from reading/listening into annotation and review creation.

Current Phase 1 progress:

- Study Guide reading blocks now launch guided writing from exact words and short phrases
- interlinear gloss view now launches guided writing from glossed words and short phrase chips
- the journal composer can accept draft-level annotation candidates and live text selections

This means HanziBit has already started building the “input to noticing to output” bridge that this strategy depends on.

---

## 3. Retrieval And Spaced Review

Long-term retention depends on active recall over time.

HanziBit already supports this through flashcards, but the product should go further.

### Better review model

- sentence-based cards, not only isolated vocabulary
- cloze cards
- phrase and grammar pattern review
- cards mined from journal entries and graded input
- audio-backed review where possible

### Product implication

Flashcards should evolve from “memory support” into “retrieval training tied to real language use.”

---

## 4. Pushed Output

Writing and speaking force the learner to retrieve, combine, and test knowledge.

The current journal is already the best place for this.

### Product implication

The journal should become a structured output engine, not only a blank page.

Examples:

- retell the passage you just read
- describe your day using this grammar point
- rewrite a sentence with a new structure
- answer a prompt using target vocabulary

This creates stronger transfer from input to active use.

---

## 5. Corrective Feedback

One of the biggest accelerators in language learning is meaningful feedback on output.

This is a major opportunity for HanziBit.

### Self-learner feedback options

- AI correction
- grammar suggestions
- naturalness suggestions
- repeated mistake detection
- rewrite suggestions at the learner’s level

### Teacher feedback options

- comments on journal submissions
- targeted corrections
- assignment-based review

### Product implication

Feedback should become a core feature, not a later add-on.

---

## 6. Daily Repetition And Routine

The app should not only provide tools. It should guide a repeatable daily practice loop.

### Product implication

HanziBit should eventually include a strong “Today” experience that answers:

**What should I do in the next 15 to 30 minutes?**

Example daily flow:

- due flashcard reviews
- short graded reading or listening
- short writing or speaking task
- feedback summary

This is more effective than expecting learners to orchestrate everything themselves.

---

## Recommended Product Priorities

If the goal is better learning outcomes, HanziBit should prioritize these areas in order.

## Priority 1: Guided Daily Learning Loop

Build a daily workflow that combines:

- review
- comprehensible input
- output
- feedback

This should become more important than adding more standalone pages.

## Priority 2: Comprehensible Input Content

Add:

- graded reading content
- short listening content
- transcript and gloss support
- content recommendations by level

This is likely the largest missing piece in the current learner experience.

## Priority 3: Journal Feedback

Add stronger output support:

- validation of markup
- guided prompts
- AI feedback
- teacher feedback later

This is where HanziBit can differentiate from pure CI tools and pure flashcard tools.

## Priority 4: Sentence-Based Review

Upgrade review from word-only memorization toward:

- sentence retrieval
- cloze exercises
- phrase and structure review
- review cards pulled from journal and input content

## Priority 5: Better Learning Metrics

Track more than HSK exposure.

Track:

- input time
- output time
- retrieval success
- recurring mistakes
- reviewed structures
- active versus passive vocabulary

This gives the learner and, later, the teacher a much stronger sense of real progress.

---

## What HanziBit Should Not Over-Prioritize Yet

If the goal is learning effectiveness, these should not come before the core loop is stronger:

- tutoring marketplace features
- broad social features
- large static reference libraries
- heavy gamification without pedagogical value
- growth layers that distract from the learner loop

These may matter later, but they should not outrank input, output, feedback, and retrieval.

---

## How This Affects `product-evolution-strategy.md`

The current product evolution strategy is directionally correct:

1. strengthen the standalone learner app
2. add classroom workflows
3. later explore marketplace ideas

However, “strengthen the standalone learner app” should be interpreted more specifically.

Phase 1 should not only mean:

- API hardening
- testing
- validation cleanup

It should also mean:

- making the learner loop more effective
- improving the journal as an output tool
- preparing the study guide to become a comprehensible input surface
- moving toward daily guided practice

---

## Recommended Updates To Phase 1

Phase 1 should explicitly prioritize:

- stable mobile and backend contracts
- journal annotation validation and usability
- guided daily review flow
- stronger input experiences
- output feedback groundwork

This keeps the technical work aligned with actual learning outcomes.

---

## Recommended Updates To Later Phases

## Phase 2

Teacher workflows should reinforce the same learning loop:

- assign input
- assign output
- review student work
- give feedback

## Phase 3

Teacher content systems should help teachers reuse:

- prompts
- readings
- vocabulary sets
- grammar explanations
- correction patterns

This will make the classroom features pedagogically stronger, not just operationally broader.

---

## Suggested Product Positioning

The best concise positioning for HanziBit is:

**HanziBit helps learners turn understandable Mandarin input into active output and long-term retention.**

That is sharper than positioning it mainly as:

- an HSK app
- a notebook app
- a flashcard app

Those are parts of the system, but not the full value proposition.

---

## Summary

If HanziBit wants to become very effective for language learning, it should optimize for a complete practice loop:

1. understandable input
2. in-context noticing
3. retrieval practice
4. pushed output
5. corrective feedback
6. repetition over time

The current app already has a strong base for output and retrieval.

The next big opportunity is to strengthen:

- comprehensible input
- guided daily practice
- feedback loops

That is the most promising path for making HanziBit significantly better at producing real language gains.
